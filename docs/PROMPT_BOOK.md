# Prompt Book

## 1. Purpose

This prompt book documents the prompt strategy used by the AI Agent Debate project. It
explains the live prompt templates, topic framing, context-window engineering, mock
behavior, provider fallback behavior, and user-requested prompt decisions made during
development.

The live templates are assembled in `src/agent_debate/memory.py`. Topic framing is
implemented in `src/agent_debate/config.py`. Mock argument and mock judge behavior are
implemented in `src/agent_debate/llm/mock_client.py`.

## 2. Prompt Design Principles

- Keep each agent role explicit.
- Keep Pro and Con stances tied to the selected topic.
- Require respectful argumentation.
- Require direct rebuttal when a previous opponent argument exists.
- Provide compact selected context, not the full transcript.
- Include web evidence candidates when search returns URLs.
- Give the judge explicit scoring criteria.
- Require a no-tie decision.
- Keep provider errors out of agent speech.
- Make mock mode deterministic enough for tests but varied enough to avoid repeated
  generic text.

## 4. Child Agent Prompt Template

Used by `PromptBuilder.child_prompt()`.

```text
You are the {role} debate agent. Stance: {stance}.
Topic: {topic}
Rules: {rules}
Running summary: {summary}
Opponent previous argument: {previous_argument_or_none}
Current judge instruction: {instruction}
Web evidence candidates: {evidence_urls}
Write one respectful turn. Explicitly rebut the opponent if present. Include source URLs you rely on.
```

## 5. Child Prompt Fields

- `role`: `pro` or `con`.
- `stance`: role-specific position from the framed topic.
- `topic`: current proposition.
- `rules`: configured debate rules.
- `summary`: compact running summary from `DebateMemory`.
- `previous_argument`: the latest opponent argument only.
- `instruction`: current parent instruction.
- `evidence_urls`: best-effort web search URLs.

## 6. Pro Agent Example

```text
You are the pro debate agent. Stance: Argue that Pilates is better than Yoga.
Topic: Pilates is better than Yoga.
Rules: {'max_words_per_turn': 170, 'require_rebuttal': True}

Opponent previous argument: Yoga is more accessible because it needs less equipment.
Current judge instruction: Round 2: present your argument and rebut the prior point.

Write one respectful turn. Explicitly rebut the opponent if present. Include source URLs you rely on.
```

## 8. Judge Prompt Template

Used by `PromptBuilder.judge_prompt()` when `final=True`.

```text
FINAL DECISION: choose exactly one winner; no tie allowed. Your answer must include these lines: Winner: <Pro Agent or Con Agent>; Score: Pro <0-100>, Con <0-100>; Reason: <one concise explanation>.
Topic: {topic}
Rules: {rules}
Round: {round_number}
Running summary: {summary}
Latest pro: {latest_pro_argument}
Latest con: {latest_con_argument}
Scoring criteria: {scoring}
Evaluate persuasion, evidence quality, factual correctness, and direct rebuttal.
```

## 9. Judge Safeguard

`JudgeAgent._ensure_final_decision()` checks whether the provider response includes:

- `Winner: Pro Agent` or `Winner: Con Agent`.
- `Score: Pro <number>, Con <number>`.

If the provider omits those fields, the system appends a required verdict generated by
the mock judge. This keeps the transcript compliant with the no-tie assignment
requirement.

## 10. Mock Prompt Behavior

`MockLLMClient` is used for tests, missing keys, and provider fallback. It does not call
an external API.

Current mock behavior:

- Generates Pro and Con turns from the selected stance and topic.
- Includes round-specific argument points.
- Has topic-specific examples for the default AI-agents topic and Pilates vs Yoga.
- Uses generic but topic-linked points for other topics.
- Scores the latest Pro and Con arguments separately for the final mock decision.
- Uses a deterministic tie-breaker only when scores are equal.

This fixes the earlier issue where mock mode produced repeated text and always selected
Pro with the same score.

## 11. Context Window Engineering

The system follows the Select/Write model from the lecture.

Write:

- Full messages are saved in `logs/debate.jsonl`.
- A readable transcript is saved in `logs/transcript.txt`.
- Exported transcripts are saved as timestamped files.
- Representative result transcripts are committed under `results/`.
- `DebateMemory` keeps full message objects for the active run.

Select:

- Child agents receive role, stance, topic, rules, opponent previous argument, compact
  summary, instruction, and evidence.
- Judge receives topic, rules, latest Pro and Con arguments, compact summary, scoring
  criteria, and round number.

## 12. Provider Behavior

Default role providers:

- Judge: Gemini through `GEMINI_API_KEY`.
- Pro: Groq through `GROQ_API_KEY`.
- Con: OpenAI through `OPENAI_API_KEY`.
- Mistral: optional through `MISTRAL_API_KEY` if config is changed.
- Mock: local deterministic fallback.

Provider errors are not inserted into agent arguments. Fallback reason is stored in
message metadata, and the CLI prints a concise notice such as:

```text
Gemini failed, so this run used mock fallback.
Provider error: Gemini quota/rate limit exceeded ...
```

## 14. Maintenance Rules

When changing prompts or prompt-related behavior:

- Update `src/agent_debate/memory.py`.
- Update `src/agent_debate/config.py` if topic framing changes.
- Update `src/agent_debate/llm/mock_client.py` if mock behavior changes.
- Update examples in this file.
- Update tests that assert prompt behavior.
- Verify that judge decisions still include a winner and scores.
- Verify context remains compact.

- Summarize provider errors.


Decision:

- Remove the earlier always-Pro behavior.
- Add tests proving Con can win when Con gives the stronger latest argument.

- Keep the parent as supervisor/router only.
- Add tests checking `pro`, `con`, and `judge` process entries.
